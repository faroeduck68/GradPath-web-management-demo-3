# 项目背景与任务目标

你现在是一位资深的全栈开发工程师。我目前正在 IntelliJ IDEA 中开发一个名为 `GradPath-web-management` 的系统。后端采用 **Java + Spring Boot**，前端技术栈为 **Vue 3 (或 React) + Tailwind CSS**。

我的目标是在该系统中实现一个**“沉浸式 AI 模拟面试”**功能。用户可以通过麦克风与 AI 面试官（Kairo）进行实时的语音对答。

## 1. 核心架构与技术栈约束

* **通信方式**：前后端必须使用 **WebSocket** 进行全双工通信，以保证极低的延迟。
* **后端 (Spring Boot)**：
  * 负责维护 WebSocket 会话和管理面试状态。
  * 负责调用 STT (语音转文字)、LLM (大语言模型)、TTS (文字转语音) 的第三方 API。
  * 需要使用 Redis 缓存会话上下文。
* **前端**：
  * 需要使用浏览器原生的 `MediaRecorder` 和 `AudioContext` 处理录音和播放。
  * 包含 VAD (Voice Activity Detection) 逻辑，识别用户说话的起点和终点。

## 2. 关键业务流程 (需实现)

1. **建立连接**：前端开启面试，通过 WebSocket 连接后端。
2. **前端采音**：用户说话，前端利用 VAD 检测到说话结束，将音频流发送给后端。
3. **后端处理管道 (Pipeline)**：
   * 接收音频 -> 调用 STT 转为文本。
   * 将文本丢给 LLM 生成回复 (要求开启 Stream 流式输出)。
   * **关键逻辑**：后端接收 LLM 的流式文本，按句子(如遇到句号、问号)进行切分，异步调用 TTS 接口生成音频片段。
4. **音频下发**：后端将生成的音频流(Base64 或二进制)以及对应的文本，实时推给前端。
5. **前端播放**：前端按顺序播放音频块，同步打字机效果显示字幕。

## 3. 难点处理要求 (请在代码中重点考虑)

* **打断机制设计**：如果前端在播放 AI 语音时用户突然说话，前端应发送 `interrupt` 信号，后端如何优雅地取消当前的生成任务并开启新一轮聆听？
* **低延迟优化**：请在后端代码中展示如何利用 Java 的异步特性 (`CompletableFuture` 或响应式编程) 来串联 STT -> LLM -> TTS 的流水线。

## 4. 你的输出任务

请分步骤帮我完成以下工作：
1. **WebSocket 协议设计**：给出前后端交互的 JSON 消息体结构 (Message Payload)。
2. **Spring Boot 后端核心代码**：提供 WebSocket Handler 的实现骨架，以及串联 AI API 的核心 Service 类。
3. **前端核心逻辑**：提供录音控制、WebSocket 消息分发和音频队列播放的核心 JS/TS 类。
